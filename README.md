# 2147483647mod

Minecraft 1.20.1 (Forge) 用Mod。
アイテムのスタック数・エンチャントレベル・ポーション効果の増幅値(amplifier)を
最大 `2147483647` (Integer.MAX_VALUE) まで設定できるようにします。

## 機能

### 1. アイテムスタック数の上限解放
- 全アイテム(バニラ・他Mod含む)のスタック上限を `2147483647` に変更します。
- クラフト結果のスタッキング、ホッパー/チェスト間の自動移動、
  クリエイティブモードでのアイテム取得数も自動的に最大値になります。
- インベントリ等の個数表示は Config (`stackCountDisplayMode`) で
  `FULL_NUMBER`(そのままの数字) / `ABBREVIATED`(1.2Bなどの省略表記) を
  切り替えられます。

### 2. エンチャントレベルの上限解放
- `/enchant` コマンド、および金床でのエンチャント本合成で
  レベル `2147483647` まで付与できます(Modエンチャント含む)。
- 効果量は各エンチャントの計算式に従い無制限にスケールします。
- レベル11以上はバニラ仕様により通常の数字で表示されます。

### 3. ポーション効果(amplifier)の上限解放
- `/effect` コマンドの増幅値引数を `2147483647` まで拡張します
  (Mod効果含む)。
- 醸造台で「ポーション + 増幅結晶」を組み合わせることで、
  ポーション内の全効果のamplifierを+1して再醸造できます。
  繰り返すことで増幅値を無制限に増やせます。
- amplifierをNBTに保存する際、バニラのbyte(-128~127)制限を
  超えても正しく保存・復元できるよう拡張しています。
- ツールチップでの増幅値表示は通常の数字で表示されます。

## Config

`config/maxint2147483647-common.toml`
- `itemMaxStackSize` (デフォルト: 2147483647)
- `enchantMaxLevel` (デフォルト: 2147483647)
- `effectMaxAmplifier` (デフォルト: 2147483647)

`config/maxint2147483647-client.toml`
- `stackCountDisplayMode` (`FULL_NUMBER` / `ABBREVIATED`, デフォルト: `FULL_NUMBER`)

## ビルド方法

```
gradlew.bat build
```

ビルド後、`build/libs/maxint2147483647-1.0.jar` が生成されます。

## 既知の制限事項

- 他Modが `Item#getMaxStackSize()` を独自にオーバーライドして
  固定値を返している場合(本Modの `Item`/`ArmorItem` への
  Mixinを経由しない実装)、その特定のアイテムのみスタック上限が
  変更されない場合があります。
- エンチャント効果量・ポーション効果量を極端に大きくした場合、
  ダメージ計算等が `int` の範囲を超えてオーバーフローし、
  予期しない挙動(負の値になる等)になる可能性があります。
- ポーション効果のHUD表示(画面左上のアイコン一覧)については
  ツールチップ表示と異なる描画経路を使用しているため、
  amplifierが非常に大きい場合に表示が崩れる可能性があります。
